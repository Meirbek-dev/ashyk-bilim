Certain Tailwind color palettes have become strongly associated with “AI-generated SaaS UI”, “template-core startup UI”, or “vibecoded” aesthetics because they are overrepresented in shadcn clones (I like shadcn, so use idiomatic shadcn ui and it's design system and tailwind variables), Vercel-inspired dashboards, and AI coding agent outputs.

The main offenders:

- `indigo-*`
- `violet-*`
- `purple-*`
- `emerald-*`
- `cyan-*`
- `sky-*`
- `fuchsia-*`

Especially combinations like:

- `bg-zinc-950 + text-white + indigo accents`
- `emerald-500` success states everywhere
- `from-indigo-500 to-violet-500` gradients
- `cyan/blue glow shadows`
- `purple pink blue` mesh gradients

Common “vibecoded” combinations:

```ts
indigo - 600 + violet - 600;
emerald - 500 + teal - 500;
sky - 500 + cyan - 400;
fuchsia - 500 + purple - 500;
```

Very recognizable patterns:

- `rounded-2xl`
- heavy blur/glassmorphism
- subtle grid backgrounds
- `border-white/10`
- `bg-zinc-900`
- gradients on every CTA
- all icons from Lucide
- all charts cyan/purple
- “AI assistant” = purple glow

Colors that currently feel less “vibecoded” and more intentional/designed:

Muted / editorial:

- `stone-*`
- `neutral-*`
- `slate-*` (carefully used)
- `amber-*`
- `orange-*`
- `rose-*` (subtle)
- `lime-*`
- `red-*` (deep/desaturated)
- `blue-*` but desaturated

More distinctive modern palettes:

- warm grays + single accent
- desaturated earth tones
- industrial palettes
- monochrome with one sharp signal color

Examples:

```ts
stone + amber
neutral + orange
zinc + red
slate + lime
warm gray + cobalt
charcoal + cream
```

A lot of high-end product design in 2025–2026 moved away from:

- saturated gradients
- neon purples
- “AI SaaS dark mode”

toward:

- restrained palettes
- typography-driven hierarchy
- subtle accents
- fewer colors overall

If your goal is to avoid “generated by Cursor + shadcn in 20 minutes” vibes:

- avoid `indigo/violet/emerald` as primary brand colors
- avoid default Tailwind saturation levels (`500/600`)
- reduce gradient usage dramatically
- use custom semantic tokens instead of raw Tailwind names
- shift hues slightly off Tailwind defaults
- use more spacing/typography hierarchy instead of color for emphasis

# Error Catalog

Date: 2026-06-21

This catalog defines stable application error codes used by the FastAPI backend, the Next.js frontend, tests, telemetry, and support workflows.

Every new `AppError` code must be added here with an owner, status, retry policy, and UI treatment.

## Rules

- Codes are stable identifiers. Do not rename a code after release; add a replacement and deprecate the old one deliberately.
- Public messages must not expose internals, stack traces, hostnames, SQL, tokens, or raw user content.
- `details` may include safe entity IDs and expected/actual state values.
- Unsafe dependency details must be logged server-side only and omitted from the envelope.
- 4xx expected user/business errors should not be reported as crashes.
- 5xx, dependency, timeout, and network errors should include request or event references.
